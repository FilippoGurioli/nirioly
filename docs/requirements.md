# Requirements

## Functional Requirements

- **FR-1** (Asset Embedding): The system shall embed the text payload of `package.conf` and the entire folder structure of the target dotfiles directory directly into the compiled binary at compile time.
- **FR-2** (Environment Extraction): Upon execution, the system shall extract the embedded dotfiles folder structure into a temporary directory (`/tmp/nirioly_dotfiles/`) in the host environment.
- **FR-3** (Package Provisioning): The system shall parse the package names from the embedded configuration and invoke the host's package manager (`pacman`) to install them using `--needed` and `--noconfirm` flags.
- **FR-4** (Configuration Linking): The system shall invoke `GNU Stow` to symlink the extracted temporary dotfiles into the user's home directory structures.
- **FR-5** (Secret Resolution): The system shall check for required sensitive strings (like passwords or tokens) by searching for a local `.env` file or inspecting the host's environment variables (e.g., `NIRIOLY_PSW`).
- **FR-6** (Subshell Isolation): The system shall intercept 100% of standard output (stdout) and standard error (stderr) streams generated by external commands (`pacman`, `stow`) and redirect them away from the active terminal interface.
- **FR-7** (Hardware-Driven Provisioning): The system shall interrogate the host hardware at runtime to detect the GPU vendor (Intel/AMD/NVIDIA) and automatically append the corresponding proprietary or open-source graphics drivers to the installation array.
- **FR-8** (Base System Bootstrap): The system shall automate the baseline Arch Linux installation from the live ISO, including setting the system clock, generating locales, configuring the keyboard layout, assigning the hostname, optimizing SSD mount options (e.g., continuous TRIM), and executing pacstrap to install the Linux kernel, sudo, a bootloader (grub), and an AUR helper (yay).
- **FR-9** (User & Privilege Provisioning): The system shall create the primary superuser account, provision home directories, and configure sudoers privileges according to the secrets passed via environment variables.
- **FR-10** (Transactional Rollback Engine): In the event of an operational failure during installation or configuration, the system shall execute a reverse-transaction sequence to safely undo modifications made to the host environment up to the point of failure.

## Non-Functional Requirements

- **NFR-1** (Portability/Self-Containment): The compiled binary must run successfully inside a standard, bare `Arch Linux ISO` live shell environment without requiring external runtime dependencies beyond standard core system utilities (`pacman`, `stow`, `sudo`).
- **NFR-2** (Telemetry/Observability): The system must persist the full unedited logs of all subshell operations to a uniquely timestamped log file inside `/tmp/nirioly/` to facilitate debugging in case of an installation failure.
- **NFR-3** (User Interface): The system must present a clean, real-time visual progress status to the terminal interface, using a progress bar and high-level status messages, ensuring the user is not exposed to raw package manager noise unless an error occurs.
- **NFR-4** (Execution Safety): If any subshell command fails (returns a non-zero exit code), the system must immediately stop all downstream operations, log a fatal error status to the UI, and point the user to the exact log file location.
- **NFR-5** (Rollback Observability & Transparency): If a rollback sequence is triggered, the system must print an explicit, itemized audit manifest to the user highlighting exactly which modifications were successfully reverted, which failed to revert, and which steps were non-reversible, ensuring no silent half-states remain unrecorded.


## Notes

Since the addition of Arch ISO configuration is a major challange I will split this project into 2 phases:

1. **Provision Phase**: the tool is supposed to be launched in an already working environment, it should just provide packages listed and configure dotfiles
2. **Bootstrap Phase**_ the tool is supposed to be launched in a completely unconfigured `Arch ISO` live shell and configure everything from that point till a working environment
