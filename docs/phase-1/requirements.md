# Requirements

## Functional Requirements

- **FR-1** (Asset Embedding): The system shall embed the text payload of `package.conf` and the entire folder structure of the target dotfiles directory directly into the compiled binary at compile time.

- **FR-2** (Environment Extraction): Upon execution, the system shall extract the embedded dotfiles folder structure into a temporary directory (`/tmp/nirioly_dotfiles/`) in the host environment.

- **FR-3** (Hardware-Driven Driver Injection): The system shall interrogate the host system metadata at runtime (e.g., parsing `/proc/fb` or running local hardware queries) to detect the active GPU vendor (`Intel/AMD/NVIDIA`) and automatically append the corresponding graphics driver package names to the installation array.

- **FR-4** (User & Privilege Provisioning): The system shall create the primary non-root user account, provision its home directory structure, and configure its administrative elevation privileges (`sudoers`) utilizing secret parameters resolved at runtime.

- **FR-5** (Package Provisioning): The system shall parse the package names from the embedded configuration (plus the injected GPU drivers) and invoke the host's package manager (`pacman` / `yay`) to install them.

- **FR-6** (Configuration Linking): The system shall invoke `GNU Stow` to symlink the extracted temporary dotfiles into the newly created user's home directory.

- **FR-7** (Secret Resolution): The system shall check for required sensitive configuration inputs (like passwords) by inspecting the host's active environment variables (e.g., `NIRIOLY_PSW`) or reading a local `.env` stream.

- **FR-8** (Subshell Isolation): The system shall intercept 100% of standard output (stdout) and standard error (stderr) streams generated by external system commands (`pacman`, `stow`, `useradd`) and redirect them away from the primary terminal UI.

- **FR-9** (Transactional User Rollback): The system shall maintain an in-memory execution log of all completed installation and configuration actions. In the event of a runtime failure, the system shall execute a reverse-transaction sequence to remove newly created user files, drop the symlinks, and uninstall newly added packages to revert the host environment to state zero.

## Non-Functional Requirements

- **NFR-1** (Telemetry & Observability): The system must persist the full unedited logs of all subshell operations to a uniquely timestamped log file inside `/tmp/nirioly/` to facilitate debugging.

- **NFR-2** (User Interface): The system must present a clean, real-time visual progress status to the terminal interface, using high-level status messages, emojis, and a smooth live-updating progress bar.

- **NFR-3** (Rollback Transparency): If a rollback sequence is triggered, the system must print an explicit, itemized audit manifest to the user highlighting exactly which modifications were successfully reverted, which failed to revert, and which steps were non-reversible.
