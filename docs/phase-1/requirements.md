# Requirements — Provision Phase

## Functional Requirements

- **FR-1** (Environment Extraction): Upon execution, the system shall extract the embedded dotfiles folder structure into a temporary directory (`/tmp/nirioly_dotfiles/`) in the host environment.

- **FR-2** (Hardware-Driven Package Injection): The system shall interrogate host hardware at runtime — reading the GPU vendor from PCI identifiers exposed under `/sys/class/drm/card*/device/vendor` (falling back to `lspci -nn` if unavailable), the chassis/form factor (laptop, desktop, or virtual machine), battery presence, wireless and bluetooth adapter presence, and CPU vendor — and shall append the corresponding conditional package groups (graphics drivers, laptop power-management tools, wireless/bluetooth utilities, CPU microcode) to the installation array.

- **FR-3** (Package Provisioning): The system shall parse the package names from the embedded configuration (plus the injected hardware-conditional packages) and invoke the host's package manager (`pacman` / `yay`) to install them.

- **FR-4** (Configuration Linking): The system shall invoke `GNU Stow` to symlink the extracted temporary dotfiles into the invoking user's home directory.

- **FR-5** (Password Resolution): The system shall resolve a single administrative password — used to authenticate privileged operations such as invoking `pacman` via `sudo` — by inspecting the host's active environment variable (`NIRIOLY_PSW`) or reading a local `.env` file.

- **FR-6** (Subshell Isolation): The system shall intercept 100% of standard output (stdout) and standard error (stderr) streams generated by external system commands (`pacman`, `stow`) and redirect them away from the primary terminal UI.

- **FR-7** (Transactional Rollback): The system shall maintain an execution journal — held in memory and persisted incrementally to disk (`/tmp/nirioly/journal-<timestamp>.jsonl`) as each step completes — of all completed installation and configuration actions. In the event of a runtime failure, the system shall execute a reverse-transaction sequence to remove the extracted dotfiles, drop the installed symlinks, and uninstall packages — restricted to only those packages that were confirmed absent from the host immediately before this run installed them; a package already present on the host before execution began must never be removed by rollback — in order to revert the host environment to state zero.

## Non-Functional Requirements

- **NFR-1** (Telemetry & Observability): The system must persist the full unedited logs of all subshell operations to a uniquely timestamped log file inside `/tmp/nirioly/` to facilitate debugging.

- **NFR-2** (User Interface): The system must present a clean, real-time visual progress status to the terminal interface, using high-level status messages, emojis, and a smooth live-updating progress bar.

- **NFR-3** (Rollback Transparency): If a rollback sequence is triggered, the system must print an explicit, itemized audit manifest to the user highlighting exactly which modifications were successfully reverted, which failed to revert, and which steps were non-reversible.

- **NFR-4** (Asset Embedding): The system shall embed the text payload of `package.conf` and the entire folder structure of the target dotfiles directory directly into the compiled binary at compile time.
